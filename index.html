I will write a paragraph for each methodology of SDLC. There are many of them but I will suggest a few. My suggestion and reasons are as follows:

  1. Agile - Agile is considered one of the best SDLC methods. It works by breaking projects into smaller parts called sprints, and making incremental changes and adjustments to the project. It is useful in real world scenario where people and teams are networking and communicating with each other which leads to productivity. The customer is also always involved in the process for feedback and suggestions so they receive a satisfactory product. And the development would focus on producing prototype rather than documentation to the customer. Since work in done in sprints it is possible for faster delivery and also due to sprints risks are lesser. It can mainly be used for projects with evolving requirements and customer oriented products.

  2. V-Shaped SDLC Model - This method is also known as Verification and Validation model. As the name says, the model aims to finally meet at a single point (like V shape) that is code the project. But before we get to coding there are steps to be taken in parallel with each other. There occurs development on one side and testing on the other side. Requirement gathering is done in parallel with acceptance testing is done to ensure system meets required needs. System design done in parallel with system testing. The designing of architecture, modules and flow of data is done and then complimentary integration test is done to ensure they work with each other components. Low level design and functionalities of each model is done and unit test done on each component to ensure its working. Then, due to all the above steps the actual coding is to go smoothly
  3. RAD - Rapid application model - Once again as the name emphasizes this model aims to quickly develop an application using iterative stages, developing prototypes and receiving feedback from users. This model aims to use pre-developed tools to aid in its process. From capturing requirements, data and developing base models and applications of the required software it is done using help of tools. AI tools, code generation tools, productivity tools are some of the many tools used. It can be used when development is to be done fast with lesser human or financial resources.
