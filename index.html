I will write a paragraph for each methodology of SDLC. There are many of them but I will suggest a few. My suggestion and reasons are as follows:

  1. Agile- Agile is considered one of the best SDLC methods. It works by breaking projects into smaller parts called sprints, user stories and making incremental changes and adjustments to the project. It is useful in real world scenario where people and teams are networking and communicating with each other which leads to productivity. The customer is also always involved in the process for feedback and suggestions so they receive a satisfactory product. And the development would focus on producing prototype rather than documentation to the customer. Since work in done in sprints it is possible for faster delivery and also due to sprints risks are lesser. It can mainly be used for projects with evolving requirements and customer oriented products.

  2. V-Shaped SDLC Model - This method is also known as Verification and Validation model. As the name says, the model aims to finally meet at a single point (like V shape) that is code the project. But before we get to coding there are steps to be taken in parallel with each other. There occurs development on one side and testing on the other side. Requirement gathering is done in parallel with acceptance testing is done to ensure system meets required needs. System design done in parallel with system testing. The designing of architecture, modules and flow of data is done and then complimentary integration test is done to ensure they work with each other components. Low level design and functionalities of each model is done and unit test done on each component to ensure its working. Then, due to all the above steps the actual coding is to go smoothly
  3. RAD - Rapid application model - Once again as the name emphasizes this model aims to quickly develop an application using iterative stages, developing prototypes and receiving feedback from users. This model aims to use pre-developed tools to aid in its process. From capturing requirements, data and developing base models and applications of the required software it is done using help of tools. AI tools, code generation tools, productivity tools are some of the many tools used. It can be used when development is to be done fast with lesser human or financial resources. This model also is flexible and can cater to user’s needs.


Among so many SDLC methodologies the ones that are in the AGILE classification are better. Among AGILE SDLC methods, I have also a few suggestions

  1. Kanban - is a Japanese word which roughly means ‘cards that can be seen.’ It uses a board called Kanban board where tasks are placed in ‘To Do’, ‘In Progress’ and ‘Done’ areas. This area is synchronized and everyone can view each card. No fixed iterations here, just pick up a task on the ‘To Do’ and get it done. It is good for continuous flow and improve efficiency. Used for project with continuous delivery, maintenance and support.
  2. Scrum - This method organizes project into short same length iterations called sprints. Specific roles like Scrum master, product owner and developers work together to get project done. Each sprint have well defined goals and there are DSMs (Daily Standup Meetings) to update progress. Tasks are prioritized based on scrum master and product owner ideas and decision. This is good for iterative projects which can be broken down to parts and take feedback on.
  3. Lean - Lean software development similar to manufacturing lean idea, it aims to remove wasted efforts, quick deliverables and optimization. It says ‘we need to deliver only what is needed, remove the rest.’ Customers and developers work together here also. It is ideal for startups and new projects with aim for efficiency and speed.
  4. FDD - Feature Driven Development - As the name says it aims on designing and building of features. The project revolves around features and is divided into different set of features and each are developed incrementally. That is why design is important here to see if each sprint of features are interconnecting with each other. It is useful for large complex projects.

I have used CSS in this webpage and there are 3 ways to include CSS in a webpage as follows:

  1. Inline - Using the ‘style’ attribute CSS are directly added to the HTML element. Example : <h1 style="color: Red; font-size: 50px;">Welcome to my homepage</h1>
  2. Embedded - Or Internal CSS, these are used withing the <style></style> HTML Tags, inside the <head> section. Example : <head> <style> h1 { color: Red; } </style></head>
  3. External - CSS present in a separate file of it’s own (usually style.css) with .css extension. Added to HTML using link tag. Example : <link rel="stylesheet" href="styles.css">


The reasons CSS used in a webpage are:

  1. Control visual aspects and look of the HTML elements.
  2. Styling and making the webpage look appealing and good design.
  3. Alignment, positioning and layout so that elements in a webpage can be correctly aligned and not mixed up and random.
  4. To maintain a consistent design and look across pages and websites by re-using CSS.
  5. Adaptability. Based on viewing device and screen size the webpage and layout must be consistent and look good.
  6. Using CSS files and re-using them is optimizing the webpage making it load faster.


We also use Javascript in webpage development. The main reasons are as follows:

  1. Main reason to use JavaScript is due to its versatility and it being the biggest and most used programming language in web development.
  2. It provides real time updates to webpages and elements without needing to reload the entire page and all it’s contents just to refresh a display value part of the webpage.
  3. It provides many interactive elements and animations.
  4. If what you need is not available in the native libraries, then there are external libraries and third part integrations that people provide and update everyday, all across the world.
  5. Validation of user inputs and data is done at client side itself with help of JavaScript, thus no wasted API calls and backend calculations.
  6. Using AJAX and Fetch APIs, it does communication with servers and backend without reloading the webpage.
  7. Several frameworks like Angular, React, Vue, are built on JavaScript and are very popular.
  8. Easy event handling and DOM (Document Object Model) manipulation.
